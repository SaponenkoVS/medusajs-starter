# Admin API Contract: Producer Management

Base path: `/admin`

All admin routes require authenticated admin access and use validated request/query schemas.

## `GET /admin/producers`

Returns a paginated producer list for the admin management route.

### Query Parameters

| Name | Type | Required | Notes |
|------|------|----------|-------|
| `limit` | number | No | Defaults to server-defined list size |
| `offset` | number | No | Pagination offset |
| `q` | string | No | Search across producer identity fields |
| `is_active` | boolean | No | Filter active/inactive producers |

### Response

```json
{
  "producers": [
    {
      "id": "prodcr_123",
      "name": "Acme Manufacturing",
      "handle": "acme-manufacturing",
      "country": "Germany",
      "website": "https://acme.example",
      "description": "Optional description",
      "is_active": true,
      "product_count": 24
    }
  ],
  "count": 1,
  "limit": 15,
  "offset": 0
}
```

## `POST /admin/producers`

Creates a producer.

### Request Body

```json
{
  "name": "Acme Manufacturing",
  "country": "Germany",
  "website": "https://acme.example",
  "description": "Optional description"
}
```

Notes:

- `handle` is generated by the system from `name`.
- If the generated handle collides with an existing producer, the system appends a unique suffix.

### Response

```json
{
  "producer": {
    "id": "prodcr_123",
    "name": "Acme Manufacturing",
    "handle": "acme-manufacturing",
    "country": "Germany",
    "website": "https://acme.example",
    "description": "Optional description",
    "is_active": true
  }
}
```

## `GET /admin/producers/:id`

Returns a single producer for edit surfaces and product-assignment display.

### Response

```json
{
  "producer": {
    "id": "prodcr_123",
    "name": "Acme Manufacturing",
    "handle": "acme-manufacturing",
    "country": "Germany",
    "website": "https://acme.example",
    "description": "Optional description",
    "is_active": true,
    "product_count": 24
  }
}
```

## `POST /admin/producers/:id`

Updates a producer.

### Request Body

```json
{
  "name": "Acme Manufacturing GmbH",
  "country": "Germany",
  "website": "https://acme.example",
  "description": "Updated description",
  "is_active": true
}
```

Notes:

- `handle` is read-only in the initial release and does not change when `name` changes.

### Response

```json
{
  "producer": {
    "id": "prodcr_123",
    "name": "Acme Manufacturing GmbH",
    "handle": "acme-manufacturing",
    "country": "Germany",
    "website": "https://acme.example",
    "description": "Updated description",
    "is_active": true
  }
}
```

## `DELETE /admin/producers/:id`

Retires or deletes a producer according to business rules.

### Response

```json
{
  "id": "prodcr_123",
  "object": "producer",
  "deleted": true
}
```

### Failure Cases

- `400`: invalid request payload.
- `409`: delete not allowed because linked products still require explicit reassignment or link removal.
- `404`: producer not found.

## `GET /admin/products/:id/producer`

Returns the currently assigned producer for the product widget.

### Response

```json
{
  "product_id": "prod_123",
  "producer": {
    "id": "prodcr_123",
    "name": "Acme Manufacturing",
    "country": "Germany",
    "website": "https://acme.example",
    "handle": "acme-manufacturing"
  }
}
```

If the product has no producer:

```json
{
  "product_id": "prod_123",
  "producer": null
}
```

## `POST /admin/products/:id/producer`

Assigns or replaces the product’s producer.

### Request Body

```json
{
  "producer_id": "prodcr_123"
}
```

Assignment rules:

- The referenced producer must exist and be active.
- Inactive producers are not valid for new assignments or replacements.

### Response

```json
{
  "product_id": "prod_123",
  "producer": {
    "id": "prodcr_123",
    "name": "Acme Manufacturing",
    "country": "Germany",
    "website": "https://acme.example",
    "handle": "acme-manufacturing"
  }
}
```

## `DELETE /admin/products/:id/producer`

Clears the product’s current producer assignment.

### Response

```json
{
  "product_id": "prod_123",
  "producer": null
}
```
